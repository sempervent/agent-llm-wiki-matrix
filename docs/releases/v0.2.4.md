# v0.2.4 — End-to-end publication, reports workflow, MkDocs

## Objective

Deliver a **coherent operator path** from **campaign run** through **validate → pack → compare → publish**, **publication-ready** campaign and compare **Markdown**, **MkDocs** documentation site with **Publication**-first navigation, **GitHub Actions** docs builds, and **reader interpretation** on pack/campaign compares—while keeping **default CI offline**.

## Scope (delivered)

- **E2E publication workflow** — **[docs/workflows/campaign-result-pack-publication.md](../workflows/campaign-result-pack-publication.md)** checklist; cross-links with **[benchmark-campaigns.md](../workflows/benchmark-campaigns.md)** and **[verification.md](../workflows/verification.md)**.
- **MkDocs** — **`mkdocs.yml`**, Material theme, **`just docs`** / **`just docs-build`**, **[docs/workflows/docs-site.md](../workflows/docs-site.md)** (handbook + findings), **`.github/workflows/docs.yml`** (strict build, artifact, optional **`gh-pages`**).
- **Compare reader interpretation** — optional **`reader_interpretation`** on **`campaign_result_pack_comparison`** and **`campaign_compare`** JSON; Markdown sections in compare reports.
- **Campaign & compare report readability** — tighter headings, digests, interpretation blocks (see **implementation log**).
- **Docs-site nav** — **Publication** block (checklist, tracking, campaigns, walkthrough, reporting pipeline, verification, longitudinal, audit); duplicate nav entries removed where noted in **docs-site**.

## Non-goals

- Dashboards, cloud runners, remote/IDE MCP, or replacing offline default CI.

## Verification

- **`uv run just ci`**
- **`just validate-artifacts`**
- **`uv run --extra docs mkdocs build --strict`**
