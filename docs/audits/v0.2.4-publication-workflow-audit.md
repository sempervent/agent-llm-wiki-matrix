# Audit: v0.2.4 publication workflow readiness

**Date:** 2026-04-19  
**Scope:** End-to-end path **campaign run → validate → result pack → pack-check → compare (packs or directories) → interpret → publish**, aligned with **[docs/roadmap/v0.2.4.md](../roadmap/v0.2.4.md)** exit themes.  
**Method:** **Evidence-driven, deterministic** — review of committed docs, tests, and inventory files only (no live providers, browsers, or network). No product code was executed for this audit beyond what is already documented in-repo.

**Related:** **[docs/tracking/v0.2.4-campaign.md](../tracking/v0.2.4-campaign.md)** · **[docs/workflows/campaign-result-pack-publication.md](../workflows/campaign-result-pack-publication.md)** · **[docs/audits/schema-drift-contracts-inventory.md](schema-drift-contracts-inventory.md)**

## Executive summary

The repository already has a **coherent operator story**: a single checklist (**[campaign-result-pack-publication.md](../workflows/campaign-result-pack-publication.md)**), a **verification matrix** (**[verification.md](../workflows/verification.md)**), **drift guards** on most committed JSON kinds (**[test_schema_drift_contracts.py](../../tests/test_schema_drift_contracts.py)**), and **committed exemplars** for pack and compare outputs. The largest **contract gap** for publication confidence is **`reports/campaign-analysis.json`**: it is **not** a registered **`alwm validate`** kind and is **excluded** from the drift sweep by explicit policy (**[schema-drift-contracts-inventory.md](schema-drift-contracts-inventory.md)** § “Not covered”). Secondary gaps are **documentation freshness** (some audit files pinned to older versions), **readability** of dense compare tables when browser evidence is present, and **operator friction** when **`just`** is not installed (fallbacks exist but are easy to miss).

## Findings

### A. Validation & contracts

1. **`campaign-analysis.json` drift** — Documented as **out of scope** for **`validate-artifacts`** (**[schema-drift-contracts-inventory.md](schema-drift-contracts-inventory.md)**). Comparative tooling and pack compare **depend** on this file’s shape; silent drift in committed examples is possible without a dedicated kind or golden snapshot test.
2. **`pack-check` vs drift** — Structural / path consistency is validated by **`alwm benchmark campaign pack-check`**, with unit coverage in **[test_campaign_result_pack.py](../../tests/test_campaign_result_pack.py)**. There is **no** CI step that runs **`pack-check`** against every committed **`examples/campaign_result_packs/*/`** tree (would be **deterministic** if added as a pytest or small script).
3. **Version metadata** — `pyproject.toml` / `alwm version` report **0.2.3** while **v0.2.4** is the active milestone; expected pre-release, but **release-readiness** and **audit** docs that still pin **v0.1.0** (**[release-readiness-audit.md](release-readiness-audit.md)**) confuse readers auditing “what is current.”

### B. Documentation

1. **Multiple entry points** — **[README.md](../../README.md)**, **[AGENTS.md](../../AGENTS.md)**, **[benchmark-campaigns.md](../workflows/benchmark-campaigns.md)**, **[campaign-result-pack-publication.md](../workflows/campaign-result-pack-publication.md)**, **[docs/wiki/benchmark-campaigns.md](../wiki/benchmark-campaigns.md)** — content is aligned in theme but **cross-link maintenance** is a recurring cost (**[verification.md](../workflows/verification.md)** explicitly calls out updating several files when **`justfile`** changes; same principle applies to workflow edits).
2. **Reproducible committed compares** — `campaign compare` exemplar uses **`--created-at`** (**[examples/campaign_compares/README.md](../../examples/campaign_compares/README.md)**). **`compare-packs`** examples emphasize **`--repo-root`**; **timestamps** in `pack-compare.json` still change on regeneration unless documented as acceptable churn.

### C. Output readability

1. **Compare reports** — Pack/campaign compare Markdown can be **wide** (e.g. browser evidence pairing tables). Usable for review, but **executive summary** / “so what” lines are **not** always first-class (tracking **[v0.2.4-campaign.md](../tracking/v0.2.4-campaign.md)** lists this as an open question).
2. **INDEX.md vs README** — Audience split (internal vs external) is noted as **open** in tracking; **no** single convention is enforced repo-wide.

| **P0** | **Refresh stale audit metadata** | Avoids wrong conclusions during release | Update **[release-readiness-audit.md](release-readiness-audit.md)** header / version table or mark **superseded** with pointer to this file and **v0.2.3** release notes. |

| **P1** | **Document timestamp / churn policy** | Reduces noise for reviewers of **`pack-compare.json`** | One paragraph in **[campaign-result-pack-publication.md](../workflows/campaign-result-pack-publication.md)** or compare README: what must be stable vs regenerated fields. |
