## Primary mission

**End-to-end publication and reports workflow** — contributors and operators can follow **one clear story** from campaign execution through **bundled, reviewable artifacts** suitable for **git** or **handoff**, with **human-readable** interpretation of **pack** and **campaign** comparisons.

---

## Priority areas

| Area | Intent |
| --- | --- |
| **Polished publication-ready campaign reports** | Campaign, comparative, and semantic Markdown (and mirrors) read as **finished** outputs: structure, headings, executive clarity, consistent terminology. |
| **Stronger end-to-end workflow docs** | Single path from **`campaign run`** → artifacts → **`pack`** → **`pack-check`** / **`compare-packs`** / **`compare`** → publication checklist; cross-link **[docs/workflows/benchmark-campaigns.md](../workflows/benchmark-campaigns.md)**, **[docs/workflows/campaign-result-pack-publication.md](../workflows/campaign-result-pack-publication.md)**, **[docs/workflows/verification.md](../workflows/verification.md)**. |
| **Report bundling and publication output quality** | What belongs in a **bundle** (reports, INDEX, analysis JSON), naming, and how outputs **compose** for readers and reviewers. |
| **Pack/campaign comparison interpretation** | **`pack-compare-report.md`**, **`campaign-compare`** Markdown, and related JSON: **scannable** deltas, plain-language **“so what”** for humans, aligned with fingerprint semantics. |
| **Operator ritual docs** | Short, repeatable **rituals**: generate → **`just ci`** / **`validate-artifacts`** → validate packs → compare → publish; gates and **when to run** optional live/smoke (see **`AGENTS.md`**). |
| **Navigable documentation site** | **MkDocs** + Material theme: **`mkdocs.yml`**, **`just docs`**, **[docs-site.md](../workflows/docs-site.md)** — surfaces E2E publication workflows without duplicating canonical Markdown sources. |

## Non-goals

- **Dashboards** or hosted analytics products.
- **Cloud deployment** or SaaS campaign runners as exit criteria.
- **Remote / IDE-hosted MCP** integrations (see **[v0.2.0](v0.2.0.md)** non-goals).
- **Replacing** offline deterministic **default CI** with live benchmarks, browsers, or providers.
