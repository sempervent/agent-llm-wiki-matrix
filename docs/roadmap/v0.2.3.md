# Milestone v0.2.3 — Result packs, fingerprint reporting, readability, drift, honest browser/MCP bounds

## Milestone theme

**v0.2.3** closes a **coherence** lap after **[v0.2.2](../releases/v0.2.2.md)**: make **campaign outputs publishable** as **result packs**, make **fingerprints** drive **reporting** (not only raw dumps), keep **reports readable**, **prevent schema/runtime drift** under default CI, and keep **browser and MCP** story **realistic** with **honest capability boundaries**.

**Status (consolidated):** The **substantive engineering and documentation goals** below are **delivered** in the tree (see **Delivered**). **Milestone closure** still requires a **tagged release** pass: **`CHANGELOG.md` [0.2.3]**, **`docs/releases/v0.2.3.md`**, and README/AGENTS “latest shipped” alignment—see **[docs/tracking/v0.2.3-campaign.md](../tracking/v0.2.3-campaign.md)** (remaining items).

**Arc context:** **[v0.2.0](v0.2.0.md)** · **Tracking:** **[docs/tracking/v0.2.3-campaign.md](../tracking/v0.2.3-campaign.md)**.

## Goals (original scope — largely satisfied)

### 1. Campaign result pack publication

- **Publication workflow** for **git-friendly** bundles: **`alwm benchmark campaign pack`**, **`campaign-result-pack.json`**, **`INDEX.md`**, mirrored layout—documented in **[docs/workflows/benchmark-campaigns.md](../workflows/benchmark-campaigns.md)** and **[examples/campaign_result_packs/](../../examples/campaign_result_packs/README.md)**.
- **Validation:** `alwm validate … campaign_result_pack`; **`alwm benchmark campaign pack-check`** (optional **`--strict`** portability).

### 2. Fingerprint-driven reporting

- **`comparison_fingerprints`** and **`campaign_experiment_fingerprints`** inform comparative and campaign **Markdown** and **JSON** mirrors; terminology aligns with **`alwm benchmark longitudinal`**.

### 3. Report readability

- Scannable **campaign-summary**, **comparative**, **semantic** outputs; reduced gratuitous duplication (see implementation log for campaign/reporting edits).

### 4. Schema / runtime drift prevention

- **JSON Schema**, **Pydantic**, **`artifacts.py`**, **`alwm validate`**, and **committed examples** guarded by **`just ci`** and **`just validate-artifacts`** / drift tests.

### 5. Browser / MCP realism with honest capability boundaries

- Default CI **offline**; **fixture-backed** browser and **partial** MCP documented; **[docs/audits/capability-classification.md](../audits/capability-classification.md)** remains the honesty bar.

---

## Non-goals

- Hosted dashboards, SaaS, or cloud campaign runners as **exit criteria**.
- **Remote / IDE-hosted MCP** or full **remote browser** protocol as a **milestone deliverable** (see **[v0.2.0](v0.2.0.md)** non-goals).
- Changing **default rubric scoring math** unless explicitly scoped in release notes.
- Replacing **Docker / Compose / Bake** as the reproducible runtime story.
- Large refactors **outside** benchmarks, campaigns, schemas, reporting, validation, browser evidence, and MCP **documentation/tests** aligned to the goals above.

## Verification commands

**Baseline (required for merge):**

```bash
uv run just ci
```

**Committed artifact contracts (JSON Schema + Pydantic, deterministic):**

```bash
just validate-artifacts
```

See **[docs/audits/schema-drift-contracts-inventory.md](../audits/schema-drift-contracts-inventory.md)** for swept filenames and explicit exclusions. Full matrix: **[docs/workflows/verification.md](../workflows/verification.md)**.

**Campaign trees and result packs (spot checks):**

```bash
uv run alwm validate examples/campaign_runs/minimal_offline/manifest.json campaign_manifest
uv run alwm validate examples/campaign_runs/minimal_offline/campaign-summary.json campaign_summary
uv run alwm validate examples/campaign_runs/minimal_offline/campaign-semantic-summary.json campaign_semantic_summary
uv run alwm validate examples/campaign_result_packs/minimal_offline/campaign-result-pack.json campaign_result_pack
uv run alwm validate examples/campaign_runs/multi_suite/manifest.json campaign_manifest
```

**Pack comparison (example):**

```bash
uv run alwm validate \
  examples/campaign_result_packs/compare_minimal_vs_multi/pack-compare.json \
  campaign_result_pack_comparison
```

**Longitudinal (offline smoke):**

```bash
uv run alwm benchmark longitudinal \
  --runs-glob 'examples/campaign_runs/minimal_offline/runs/*/manifest.json' \
  --out-dir /tmp/alwm-long-v023-smoke
```

**Prompt registry (when registry work is in scope):**

```bash
uv run alwm prompts check
```

Optional integration (not default CI): live benchmarks, Playwright—see **`AGENTS.md`**.
