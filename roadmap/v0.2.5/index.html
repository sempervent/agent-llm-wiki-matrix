    Milestone v0.2.5 — Publication-quality evidence packs and final report readability¶

    Milestone theme¶

    v0.2.5 follows v0.2.4 (E2E publication workflow, MkDocs, compare interpretation): tighten presentation of evidence packs and reports for reviewers, refine comparison UX, reduce duplication in generated Markdown, and keep validation and docs-site navigation ergonomic—without changing default offline CI.

    Status: Active — current in-flight milestone. Tracking: docs/tracking/v0.2.5-campaign.md.

    Arc context: v0.2.0 · Prior release: v0.2.4.

    Primary mission¶

    Polish publication-quality evidence packs and final report readability — packs and campaign outputs read as finished artifacts; compare workflows stay scannable; drift and validation remain low-friction; the MkDocs site stays easy to navigate.

    Priority areas¶

    Area Intent
    Publication-ready pack presentation INDEX, snapshots, and mirrored layout read clearly for cold reviewers; consistency with campaign-result-pack-publication.md.
    Comparison workflow refinement compare-packs / compare outputs: structure, interpretation, less noise.
    Report polish and deduplication Campaign, comparative, semantic, and compare Markdown: fewer redundant blocks; clearer executive vs detail.
    Drift-prevention and validation ergonomics validate-artifacts, alwm validate, and docs for when to run what; optional inventory updates.
    Docs-site polish and navigation quality mkdocs.yml nav, docs-site.md kept accurate; home page discoverability.

    Non-goals¶

    • Dashboards or hosted analytics.
    • Cloud deployment or SaaS runners as exit criteria.
    • Remote / IDE-hosted MCP (see v0.2.0 non-goals).
    • Replacing offline deterministic default CI with live benchmarks or providers.

    Exit criteria (draft)¶

    Area Target
    Packs & compares Committed examples reflect polished presentation; compare reports pass a readability bar agreed in tracking.
    Reports Reduced gratuitous duplication; headings and sections intentional.
    Validation story verification.md and docs-site stay aligned with justfile.
    Docs site mkdocs build --strict green; nav updated when new primary workflow pages ship.
    Contracts uv run just ci and just validate-artifacts green.

    Dependencies¶

    • v0.2.4 baseline (publication workflow, MkDocs, reader interpretation).

    Verification commands¶

    Baseline: uv run just ci · just validate-artifacts · uv run --extra docs mkdocs build --strict

    Full matrix: docs/workflows/verification.md · handbook docs/workflows/docs-site.md.
